## 📋 Workflow Chi Tiết

### STEP 1: Detect CR

- Tìm mã CR từ user message (pattern: `LCR\d+-\d+`)
- Nếu không có → hỏi user chọn từ danh sách đã biết
- Nếu user nhập ticket Epic/Story → resolve về mã CR

### STEP 2: Detect Username

- Mặc định: `khangld10` (từ MCP config X-Member-Id)
- Hoặc hỏi user confirm

### STEP 3: Tổng hợp Tasks

Scan conversation history:
- Tickets đã làm việc
- Actions: viết test, log bug, review, tạo automation, v.v.
- Kết quả: pass/fail, số lượng

### STEP 4: Hybrid Time Tracking

| Loại task | AI time | Manual estimate |
|-----------|---------|-----------------|
| Tạo test plan (30 TC) | ~40p | ~3h |
| Gen test case (10 TC) | ~15p | ~1h |
| Log bug + phân tích | ~10p | ~30p |
| Viết spec/requirements | ~30p | ~2h |
| Setup Confluence page | ~10p | ~45p |
| Viết automation test | ~40p | ~3h |
| Tạo data test | ~15p | ~1h |
| Verify test cases | ~20p | ~2h |
| Publish lên Confluence | ~5p | ~20p |

Nếu task không có trong bảng → Manual = AI time × 3~5

### STEP 5: Preview & Confirm

Hiển thị cho user:
- Tasks grouped theo CR
- AI time + Manual estimate cho mỗi task
- Tổng ngày
- Hỏi: A) Publish / B) Chỉnh sửa / C) Không publish

### STEP 6: Publish

1. Tìm page chính CR (dùng page_id đã biết hoặc search)
2. Gọi MCP API `rde_publish_to_confluence`:
   - title: `[AIN] 📆 DD/MM/YYYY — Daily Progress — LCR25-XXXX`
   - parent_page_id: page_id của page chính CR
   - space_key: AIN
3. Nếu page đã tồn tại → đọc nội dung cũ → append dòng mới → publish lại
