📦 Bước 1: Setup engine (1 lần duy nhất)

Paste prompt này vào AI agent (Kiro / Cursor / Claude Code):

Setup engine recorder cho tôi. Workspace folder: ~/work

Agent tự clone repo, install, link global command, và cài playbook vào steering của workspace. Xong trong ~2 phút.

Hoặc clone thủ công (dùng HTTPS):

# Clone qua HTTPS (KHÔNG dùng SSH) — nhập user/password hoặc Personal Access Token khi được hỏi
git clone https://git2.fptshop.com.vn/ai-test/recorder-engine.git ~/work/recorder-engine
cd ~/work/recorder-engine
npm install
npx playwright install chromium
npm link
🔑 Vì sao HTTPS? Không cần cấu hình SSH key. Đăng nhập bằng tài khoản GitLab; nếu bật 2FA thì tạo Personal Access Token (GitLab → Settings → Access Tokens, scope read_repository) và dùng token đó thay password.

✅ Done! Từ đây bạn đã có thể record/play/regression theo từng env. Không cần config thêm gì.

🎬 RECORD TEST — CHI TIẾT

Khi bạn paste prompt record, đây là những gì xảy ra:

1
Browser Chromium mở lên
App URL tự động load. Bạn thấy browser hiện ra.
2
Bạn thao tác bình thường
Click, nhập liệu, chọn dropdown, navigate... Hệ thống ghi lại mọi thứ.
3
Thêm assertion (kiểm tra kết quả)
Ctrl+Shift+A → click element → nhập giá trị mong đợi. Đây là cách verify kết quả.
4
Ctrl+C để dừng
File YAML tự động lưu vào flows/{app}/{env}/{tên-test}.yaml (vd flows/rsa-lab/ci/tao-don-hang.yaml)
⚠️ Mẹo record tốt:
• Thao tác chậm, rõ ràng — đợi page load xong rồi mới click
• Sau khi chọn dropdown → đợi 1-2 giây trước khi click tiếp
• Hover để hiện nút: nếu nút chỉ hiện khi rê chuột vào (icon/row) → cứ hover rồi click, recorder tự ghi bước hover
• Popup chắn đường: popup/quảng cáo bất ngờ sẽ được tự đóng khi play — không cần click tắt thủ công lúc record
• Đặt tên test có ý nghĩa: "tao-don-hang-dv-xn-mau" thay vì "test1"
💡 Assert giá trị động (tiền, tồn kho, %, ngày):
Khi dùng Ctrl+Shift+A để verify một giá trị hay đổi (vd "1,234,000đ", "Tồn 21", "12.5%") → recorder tự chuyển sang kiểm tra element có hiện không thay vì so khớp con số. Nhờ vậy test không fail khi data đổi giữa các lần chạy.

🔧 TEST FAIL? AI TỰ SỬA

Khi test fail (selector lỗi, DOM thay đổi, timing...), chỉ cần 1 prompt:

Test "tao-don-hang" của LAB chạy fail. Fix giúp tôi.

Agent sẽ tự:

  • Chạy lại test, đọc error message
  • Xem screenshot fail
  • Phân tích nguyên nhân (selector cũ? timing? URL đổi? thiếu bước hover? popup chắn?)
  • Sửa file YAML trực tiếp (phần thao tác)
  • Chạy lại verify các step thao tác chạy hết
  • Báo bạn kết quả + hỏi commit không
⛔ Nguyên tắc khi fix: KHÔNG sửa assertion để ép test pass.
Agent chỉ fix để flow chạy được đến cuối (selector, timing, hover, popup, navigation). Nếu một assertion fail → đó thường là BUG sản phẩm mà test bắt được. Agent sẽ giữ nguyên assertion + báo bug, KHÔNG đổi giá trị mong đợi cho khớp để "xanh giả".

Ngoại lệ: nếu assertion so khớp dữ liệu động (tiền/tồn kho/%/ngày — thay đổi mỗi lần chạy, không phải bug) thì mới đổi sang kiểm tra visible hoặc tham số hóa {{var}}.
Bạn không cần đọc code YAML. Agent hiểu context và tự fix lỗi thao tác. Nếu là bug thật, nó báo bạn thay vì giấu đi.

Lỗi Nguyên nhân Fix

Cannot find module 'tsx' Chưa install dependencies cd ~/work/recorder-engine && npm install
Browser không mở khi record Playwright chưa cài Chromium npx playwright install chromium
Permission denied khi npm link npm global dir cần quyền sudo npm link hoặc fix npm prefix
git push rejected Repo GitLab chưa empty Xóa README mặc định trên GitLab
Test timeout Page load chậm Thêm timeout: 60000 trong YAML
Selector not found DOM đã thay đổi Dùng prompt "fix giúp tôi" → agent tự sửa
💡 MẸO HAY
  • Shared flows: Login, chọn cửa hàng... đặt trong _shared/ — các test khác use: lại, DRY
  • Tags: Gắn tag cho test (smoke, regression, critical) → chạy theo nhóm
  • Headless: Regression chạy headless (không mở browser) — nhanh hơn, CI/CD friendly
  • HTML report: Chia sẻ kết quả đẹp cho PM/lead xem
  • Thêm app: Chuỗi có app mới? 1 prompt "Thêm app..." là xong
  • Update engine: Team maintain engine → bạn chỉ git pull là có version mới
  • Assert mode: Ctrl+Shift+A khi recording = thêm verification point — quan trọng để phát hiện bug!

# CLI Commands (dùng trực tiếp nếu không muốn qua AI agent)
# Chuỗi multi-env (schema v2) cần --env. Chuỗi cũ (v1) bỏ qua được.

frt-test play --app rsa-lab --env ci tên-test.yaml # Play 1 file
frt-test play --app rsa-lab --env ci --all # Play all 1 app/env
frt-test play --app rsa-lab --env ci --all --headless --html-report # Regression
frt-test list --app rsa-lab --env ci # List tests 1 app/env
frt-test envs # Xem envs + URL status
frt-test add-env --app rsa-lab --env uat --url https://... # Set URL cho env
frt-test clone-flow tên-test.yaml --app rsa-lab --from ci --to uat # Clone test giữa env
frt-test migrate-config --write # Migrate v1 → v2 multi-env
frt-test scaffold-chain --name lab --apps "..." --envs ci,uat,prod # Tạo chain mới
frt-test add-app --name app-x --urls "ci:https://..." # Thêm app
frt-test convert input.spec.ts # Convert Playwright → YAML
frt-test --version # Check version
