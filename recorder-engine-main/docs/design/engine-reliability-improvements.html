Engine Reliability Improvements

SDD — 2026-06-04 | Plan: docs/plans/engine-reliability-improvements.html

Context

FRT Test Recorder engine có 2 vấn đề ảnh hưởng reliability khi replay recorded flows:

  • Issue #1: Mỗi step (click/fill/selectOption/keyboard) mặc định gắn delay: 1000 — gây chậm vô ích vì runner đã tự wait element.
  • Issue #3: Assert mode record text literal cho giá trị volatile (tiền, %, ngày) → flaky khi data thay đổi.

Requirements

  • Xóa delay: 1000 khỏi 4 case trong actionToStep(). Giữ field delay trong type (backward compat).
  • Mở rộng isVolatileText() nhận thêm: currency ($, €, USD, EUR), percentage (%), date (dd/mm/yyyy).
  • Khi assert mode chọn type=text mà value volatile → auto-downgrade sang assertType: 'visible' + log debug.

Design Decisions

Issue #1: Remove hardcoded delay

  • Surgical removal — chỉ xóa delay: 1000, không thay bằng logic khác.
  • Runner đã có waitFor, waitAfter, và auto-wait của Playwright → không cần padding.
  • Old flows đã save delay: 1000 trong YAML vẫn hoạt động (runner vẫn đọc field).

Issue #3a: Volatile text patterns

  • Thêm 3 regex mới vào isVolatileText(), chèn TRƯỚC rule \d{3,} (ưu tiên semantic match).
  • Date pattern dùng character class [/\-.\] cover separator phổ biến (/, -, ., \).
  • 04/06/2026 cũng match bởi rule \d{3,} (year=2026), nhưng date regex cho semantic rõ hơn.

Issue #3b: Auto-safe volatile assert

  • Dùng biến effectiveType thay vì mutate chosenType — giữ original choice cho debug nếu cần.
  • Log format: __FRT_RECORD_DEBUG__: [auto-safe] volatile assert text → changed to visible: "value"
  • Chỉ affect chosenType === 'text' — các type khác (count, containsAll, visible) không bị ảnh hưởng.

Issue #2: Auto-dismiss Popups/Modals

Design Decisions

  • Config field: autoPopupDismiss?: boolean in FlowConfig — opt-out pattern (default true). Set false when testing popup behavior itself.
  • Injection point: Second addInitScript after existing OneSignal dismisser. Same pattern: MutationObserver on document.body.
  • Throttle: 300ms timestamp-based throttle avoids performance overhead from rapid DOM mutations.
  • Safety guard: Danger words regex (/xóa|delete|xác nhận xóa|confirm delete/i) prevents accidental data destruction. If only dangerous buttons found → skip entirely.
  • Close button priority: [aria-label*="close"] → [class*="close"] → text match (×, ✕, Đóng, Close, Cancel, Bỏ qua, Huỷ).
  • Auto-disappearing elements: .ant-notification-notice, .ant-message-notice are skipped (they auto-dismiss themselves).
  • Modal mask fallback: If no safe close button found for .ant-modal-mask/.ant-modal-wrap, click mask itself (standard Ant Design dismiss-by-clicking-outside).
  • Recorder skip: Backdrop/mask clicks filtered in pointerup handler — avoids recording useless "click overlay" steps that break replay.
  • Cleanup: Observer disconnects on beforeunload to prevent memory leaks across navigations.

Issue #4 — Hover-to-Reveal Detection

Context

Hidden action buttons (edit/delete icons) only appear on row hover. Recorder missed these → playback clicks fail because target never becomes visible. Runner already supports action: 'hover' — recorder just needed to emit it.

Design Decisions

  • Auto-detect over manual hotkey: Chose auto-detection (mouseover tracking + ancestor heuristic) because it requires zero user awareness. User just hovers naturally; recorder infers the intent. A manual hotkey would be forgotten and adds cognitive load.
  • Debounce 150ms: Prevents capturing every child element as mouse traverses. Only the final resting element is stored as hover target.
  • 1500ms hover window: If click happens within 1.5s of the last hover, check relationship. This accommodates CSS transition delays and user think time.
  • Ancestor heuristic: Only emit hover when the hover target is an ancestor of the raw click target (hoverEl.contains(rawTarget)). This covers the pattern: hover on row → hidden button inside row appears → click button.
  • Runner enhancement: Added waitFor({ state: 'visible' }) before hover (ensures element exists) and waitForTimeout(300) after (allows CSS transitions to reveal children).
  • actionToStep: Handles role/text/selector locator types for hover steps, generates descriptive step names.

Autonomous Decisions

  • Auto-detect over manual hotkey: Chose auto-detection because it's invisible to the user — no extra key to remember. Hover-reveal is a common UI pattern (table row actions, icon tooltips) and should "just work" without user intervention.
  • Hover emitted only when ancestor relationship exists: Conservative heuristic — only emits hover step when the hovered element contains the subsequently clicked element. This avoids false positives from unrelated hover→click sequences on the same page.
  • No new types added: Verified types.ts — 'hover' already exists in the FlowStep action union. No RecordedAction changes needed since the payload goes through the generic __frtRecord binding.
