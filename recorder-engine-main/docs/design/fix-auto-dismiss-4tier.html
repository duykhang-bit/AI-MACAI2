Fix Auto-Dismiss — 4-Tier Model

Software Design Document · Stage 3 of 3 · 2026-06-05

Context

Problem

The original autoPopupDismiss: true used a single aggressive selector list that included .ant-modal-wrap and .ant-modal-mask. This caused the runner to auto-close legitimate functional modals (forms, confirm dialogs with data) mid-flow, breaking test reliability.

Root Cause

No distinction between junk popups (notifications, cookie banners, ad overlays) and functional modals (forms, tables, combobox dropdowns). A single boolean config blurred this distinction.

Solution

Introduce a 4-tier popup mode replacing the boolean: 'off', 'junk-only' (default), 'aggressive', plus boolean backward-compat aliases (true → aggressive, false → off). The default changes from aggressive to junk-only, protecting all modals unless explicitly opted in.

Design Decisions

4-Tier Mode Mapping

undefinedjunk-onlyDismiss notifications/banners only; never touch modals

Browser / Node Context Boundary

All constants live in src/popup-dismiss.ts (Node-side canonical). Browser-side logic receives values via addInitScript args — never imports Node modules. The IS_FUNCTIONAL_MODAL_INLINE string is a serialized function passed to the browser context.

FUNCTIONAL_MODAL_SELECTOR is the single source of truth for the form-detection CSS selector. Both the Node-side isLikelyFunctionalModal and the browser-side IS_FUNCTIONAL_MODAL_INLINE must embed this exact string — enforced by unit tests (Group 6).

Reactive Dismiss (tryReactiveDismiss)

Called before every force-retry in click/fill handlers. Uses page.evaluate to inspect overlays at the viewport center. Returns one of: none | closed | protected | functional | unknown. Protected and functional modals throw descriptive errors; unknown falls through to force-retry (preserving combobox behavior).

Backward Compatibility

Existing flows with autoPopupDismiss: true continue to work unchanged (mapped to 'aggressive'). The default for flows with no setting changes from aggressive to junk-only — a safer default that is unlikely to break correctly-written flows.
